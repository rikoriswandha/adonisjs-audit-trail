## Considered and rejected (do not re-audit)

- **Provider has no shutdown/flush hook** — false. `providers/audit_provider.ts:113-118`
  already calls `pipeline.shutdown(5000)` and `outboxDrainer.stop()`.
- **CI can silently skip the DB matrix / concurrency proofs** — false. `tests/helpers/containers.ts:34-42`
  returns `null` only when `SKIP_DOCKER_TESTS=1`; otherwise `startPostgres`/`startMysql`
  throw on Docker failure, and the integration job sets `SKIP_DOCKER_TESTS=0`. CI is fail-closed.
- **HTTP store SSRF** — false. The URL is operator config (`HttpStoreOptions.url`), not
  request-derived user input.
- **Timing-unsafe HMAC comparison in HTTP store** — out of scope. `http_store.ts` only
  *signs* outgoing batches; it performs no signature verification.
- **Pipeline drops events silently on shutdown** — false. `pipeline.ts:158-163` routes
  undrained events to the dead-letter handler and emits `audit:dead_letter`.
- **`luxon` should not be a peer dep** — not worth doing. It is type-only here, but
  `@adonisjs/lucid` (a required peer) already depends on `luxon`, so consumers always have it.
- **`engines.node: ">=24"` is too strict** — deferred to maintainer. Deliberate posture
  choice, not a defect. Surfaced as a direction item only.
- **Redactor unbounded recursion** — speculative. Audit values come from model attributes,
  not adversarial deeply-nested JSON. Not planned.
